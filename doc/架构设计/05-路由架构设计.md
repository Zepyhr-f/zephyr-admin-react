# 前端路由架构设计

> 本文档定义后台管理系统的前端路由体系、权限路由治理、懒加载策略以及路由守卫实现，确保页面访问安全、快速且易于维护。

---

## 1. 路由层级与分组

```
routes/
├── index.tsx            # 主入口，合并所有路由模块
├── public-routes.tsx    # 公开路由（登录、注册、忘记密码、错误页）
├── admin-routes.tsx     # 鉴权路由，基于后端菜单动态注册
├── auth-guard.tsx       # 路由守卫组件（登录检查、权限校验）
└── route-meta.ts        # 路由 meta 类型定义
```

### 1.1 公共路由（Public Routes）
- **路径**：`/login`, `/forgot-password`, `/403`, `/404`, `/500`
- **特点**：不需要 Token，直接渲染对应页面。

### 1.2 鉴权路由（Admin Routes）
- **路径**：`/dashboard`, `/system/*`, `/biz/*` 等业务模块。
- **入口**：`<AuthGuard>` 包裹，统一进行登录校验与权限检查。
- **懒加载**：使用 `React.lazy` + `Suspense` 按路由分块加载，配合 Vite 的 `import()` 动态导入实现代码分割。

## 2. 路由 Meta 信息规范

| 字段 | 类型 | 说明 |
|---|---|---|
| `title` | `string` | 页面标题，统一写入 `<title>`，支持 i18n 翻译键。
| `icon` | `string` | 左侧菜单图标，使用 Iconify 标识。
| `keepAlive` | `boolean` | 是否开启页面缓存（`react-keep-alive` 实现）。
| `requiresAuth` | `boolean` | 是否需要登录（默认 `true`）。
| `permission` | `string[]` | 权限标识数组，后端返回的 `perms` 字段。
| `breadcrumb` | `string[]` | 面包屑层级，自动生成。

在路由配置中必须提供 `meta` 对象，示例：
```tsx
export const systemRoutes = [
  {
    path: '/system/user',
    element: lazy(() => import('@/pages/system/user')),
    meta: {
      title: 'system.user.title',
      icon: 'mdi:user',
      permission: ['sys:user:list'],
      keepAlive: true,
      requiresAuth: true,
    },
  },
];
```

## 3. 动态路由注册

后端 `zephyr-gateway` 提供 **菜单权限接口** `/api/menu/tree`，返回用户可见的菜单树结构。前端在登录成功后通过 `useQuery` 拉取该数据并生成路由。

```tsx
const { data: menuTree } = useQuery('menuTree', fetchMenuTree, {
  onSuccess: (tree) => {
    const routes = generateRoutesFromMenu(tree);
    setDynamicRoutes(routes);
  },
});
```

`generateRoutesFromMenu` 的核心步骤：
1. 遍历菜单节点，映射 `path`、`component`（约定 `@/pages/${module}/${page}`）
2. 合并到 `adminRoutes` 中
3. 使用 `useRoutes` 渲染

## 4. 路由守卫实现（AuthGuard）

```tsx
import { Navigate, useLocation } from 'react-router-dom';
import { useAuthStore } from '@/store/use-auth-store';

export const AuthGuard: React.FC<{ children: ReactNode }> = ({ children }) => {
  const { token, userPermissions } = useAuthStore();
  const location = useLocation();

  // 未登录
  if (!token) {
    return <Navigate to="/login" state={{ from: location }} replace />;
  }

  // 权限检查（路由 meta 中的 permission）
  const route = useMatch(location.pathname);
  if (route?.meta?.permission?.length && !route.meta.permission.every(p => userPermissions.includes(p))) {
    return <Navigate to="/403" replace />;
  }

  return <>{children}</>;
};
```

**要点**：
- **Token 刷新**：拦截 401 错误后自动调用 `/auth/refresh`，成功后重新设置 token 并继续请求。
- **懒加载占位**：`<Suspense fallback={<PageLoading />}>` 包裹子路由，提升首屏感知速度。

---

## 5. 路由权限缓存

- 登录成功后，`userPermissions` 会保存到 `localStorage`（`useAuthStore` 中的 `persist` 中间件），提升刷新后首次路由匹配速度。
- 每次进入页面前，会先检查本地缓存的权限，如缺失或过期则重新请求后端。

## 6. 代码分割与预加载

| 场景 | 实现方式 |
|---|---|
| **主路由**（dashboard、系统设置） | `React.lazy` + Vite `import()`，生成独立 chunk |
| **子模块**（用户管理） | 在 `systemRoutes` 中使用 `lazy(() => import('@/pages/system/user'))` |
| **高频页**（首页、个人中心） | 使用 `webpackPrefetch: true`（Vite `import(/* webpackPrefetch: true */ ...)`）实现预取 |
| **低频页**（日志审计） | 使用 `webpackPreload: false`，仅在用户点击时加载 |

---

## 7. 常见错误处理

1. **路由循环**：确保 `generateRoutesFromMenu` 不会产生重复 `path`，使用 `Set` 去重。
2. **权限失效**：Token 失效后，统一在 `axios` 响应拦截器中抛出 `401`，由 `AuthGuard` 捕获并跳转登录。
3. **懒加载报错**：在 `Suspense` fallback 中加入错误边界 `ErrorBoundary`，防止网络异常导致页面白屏。

## 8. 迁移与演进

- **从 Vite 2 升级到 Vite 6**：仅需更新 `vite.config.ts` 的 `resolve.alias` 配置，保持 `@/` 别名不变。
- **新增动态路由字段**：如 `meta.permission` 增加 `anyOf` 组合，需同步后端返回结构。
