## 阅读建议
- 新同学先看《项目技术栈与结构规范》建立前端边界。
- 编码前补看《编程质量与 Clean Code 规范》和《React 与 TypeScript 最佳实践》。
- 涉及提交、分支和协作方式时，查看《Git 工作流与提交规范》。

## 维护规则
- 本目录只维护前端开发规范，不替代架构设计中的长期结构说明。
- 工具链、目录边界和协作约束变化时，应同步检查快速开始和前端架构文档。
- 版本口径与环境要求变化时，以版本矩阵为准并保持引用一致。
